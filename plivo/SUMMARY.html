The Flaky Network: Architecture Summary

Pipeline: Interleaved Piggybacked FEC Engine

1. System Landscape & Requirements

The core objective is engineering a high-reliability transport scheme across a hostile network layer prone to random packet drops, jitter spikes, and packet reordering[cite: 1]. Strict operational boundary metrics require a deadline-miss rate <= 1% while bounded safely within a maximum <= 2.0x streaming bandwidth capacity budget[cite: 1]. The primary optimization target is the reduction of terminal playout delay[cite: 1].

2. Architecture: Interleaved Piggybacked FEC

To avoid the fatal real-time delay costs associated with feedback loops (ARQ) or network transmission overhead skips[cite: 1], this iteration leverages a Piggybacked Forward Error Correction (FEC) scheme[cite: 1].

Rather than multiplying independent packet streams across interfaces, the sender dynamically combines the active frame payload and the immediate historical frame payload into a single, cohesive compound packet layout[cite: 1]. This packs exactly two full 164-byte payload spaces into a single 328-byte datagram buffer[cite: 1].

3. Strategic Operational Advantages

  • Deterministic Overhead Protection: Because every datagram strictly encapsulates two sequential frames, the network load maps perfectly onto the 2.0x budget limit without demanding tricky scheduling logic or skipping patterns[cite: 1].
  • Stateful Frame Erasure Recovery: If a network hazard drops packet N entirely, the arrival of packet N+1 instantly passes the missing packet history data directly into the receiver loop for immediate playback restoration[cite: 1].
  • Minimized Kernel Interrupt Overhead: Consolidating data into unified compound frames drops the active transmission frequency in half relative to naive multi-packet blast methodologies, reducing CPU scheduler thread wakes.

4. Receiver State Machine Execution

The receiver tracks sequence counters sequentially[cite: 1]. Upon receiving a compound frame, it calculates whether a frame gap occurred by checking the backup frame data window[cite: 1]. If an item was missing from the stream, it isolates the historical chunk and dispatches it immediately to the player before processing the current frame space, ensuring smooth playout continuity[cite: 1].
